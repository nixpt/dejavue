# Dejavue v0.1 scope

Filed s156 by foreman after captain ratified: park joker-mcp embedder bug, ship dejavue v0.1 today using FTS5/keyword recall.

Companion to:
- `context.md` (chatgpt spec)
- `continued.md` (chatgpt evolution: semantic, boot packet, since/why/recall/etc.)

---

## North-star principle

**Dejavue v0.1 is the zero-ceremony on-ramp to project-local memory for non-Joker repos.** It must work in 5 seconds with no infrastructure beyond `git` and a Python 3 install. Semantic recall is optional, off by default in v0.1, and added behind a flag in v0.2 once the joker-mcp embedder thread closes.

The four anchors:
1. **Repo-local, portable** — `.dejavue/` directory ships in the git repo.
2. **Hook-driven, not voluntary** — git `post-commit` does the recording.
3. **`since` is the killer command** — temporal delta over timeline + git.
4. **Degrades to FTS5 / sqlite-fts when embedder is down** — never blocks on the embedder.

---

## Architecture ruling (s156)

**Dejavue is the file format + reference CLI. Joker consumes dejavue. They are NOT bundled.**

Analogy: `.dejavue/` is to project memory what `.git/` is to history. Multiple tools sit on top of the same canonical on-disk format.

```
dejavue (portable, canonical)
  .dejavue/      = on-disk format (timeline.jsonl + state.md + decisions.md + handoff.md + fts.db + references/)
  dejavue CLI    = reference implementation
  Zero MCP dep   = drops into any repo

joker-mcp (rich consumer, separate project)
  reads/writes .dejavue/ via thin shim tools (v0.3)
  ALSO maintains .jagent/ (planning/milestones/phases/learning.db — superset)
  joker.dejavue.* MCP tools = wrappers over the file format, not parallel storage

other MCP-aware tools (Cursor, Aider, opencode, codex-cli, claude-cli)
  read .dejavue/ independently — format is the open contract
```

**Why not bundle:** bundling re-couples zero-ceremony with infrastructure, kills portability across agent ecosystems, forces joker install for basic memory. Joker-as-consumer keeps dejavue portable AND keeps joker optional.

**Migration path:**
- **v0.1** — dejavue standalone Python CLI. `.dejavue/` and `.jagent/` coexist harmlessly in same repo.
- **v0.2** — dejavue `--semantic` flag calls joker embedder when present, FTS5 fallback when absent. Embedder thread (`FOREMAN_THREADS.md`) must close first.
- **v0.3** — joker-mcp gains `joker_record_dejavue_event` / `joker_get_dejavue_context` MCP tools reading the same `.dejavue/` artifacts. Foreign-tool interop story established.
- **v0.4** — `dejavue migrate-to-jagent` upgrade tool when a repo gets serious. Reverse not needed (`.dejavue/` is subset of `.jagent/` content).

## Adoption from `ctx` (squadron's `~/.local/bin/ctx`)

`ctx` is workspace-scope curated reference docs at `.squad/ctx/`. Dejavue is repo-scope auto-captured events. Different jobs, no merge — but absorb 4 patterns:

| ctx command | dejavue v0.1 absorption |
|---|---|
| `ctx get <path>` | `dejavue get <doc>` — direct fetch without recall search (state, handoff, decisions, references/*). Friendlier than tail-piping context. |
| `ctx list` | `dejavue list [--type events\|decisions\|references]` — print available content with paths. Tab-completion friendly. |
| `ctx annotate <path> "note"` | `dejavue annotate <doc> "note"` — append timestamped note without rewriting. Clean fit for evolving handoff/state across multiple events in same session. |
| (curated doc store under `.squad/ctx/`) | `.dejavue/references/` subdir for hand-written reference cards. Recall hits both timeline + references. |

**Explicitly skip from ctx:**
- ❌ `parse` / `index` / `refs` / `def` (CAST AST code intel) — dejavue is cognitive memory, not code structure. Code intel belongs in a separate tool.
- ❌ `remember` / `recall` / `memory` (agent-mem wrapper) — already covered by `squadron_mem_*` and joker tools. Dejavue uses its own sqlite FTS5; don't introduce a third memory wrapper.

**Long-term**: after v0.3 lands and `joker_record_dejavue_event` exists, re-evaluate whether `.squad/ctx/` should become a workspace-level `.dejavue/` (`dejavue --workspace` mode). Not now — premature merge damages both.

## Non-goals for v0.1

Explicitly *not* in v0.1, to avoid the duplication trap:

- ❌ MCP server / `joker.dejavue.*` tools — wait for v0.3.
- ❌ Semantic embeddings — wait for v0.2 when joker-mcp embedder is fixed.
- ❌ Cross-repo memory federation — that's `.squad/` and FOREMAN_STATE.
- ❌ Replace MEMORY.md / `.jagent/` / `.squad/` — dejavue is the *precursor* to `.jagent`, not a parallel system.
- ❌ Auto-summarize commits via LLM — adds dependency on a chat model. v0.1 records the diff stat + commit message verbatim; LLM summarization is v0.2.
- ❌ Auto-detect "important" decisions — agents/devs explicitly mark with `dejavue decision`. Worthiness gate is enforced by the human, not heuristics.

## Directory layout

```
.dejavue/
  timeline.jsonl       # append-only event log (one JSON event per line)
  state.md             # current state summary (overwritten on dejavue state)
  decisions.md         # append-only architectural decisions
  handoff.md           # latest handoff (overwritten on dejavue handoff)
  references/          # hand-written reference cards (ctx-style curated docs, optional)
    <name>.md          # e.g. references/architecture.md, references/api-contracts.md
  ingested.lock        # marker: dejavue ingest already ran; don't re-scrape
  fts.db               # sqlite FTS5 index over timeline + decisions + state + handoff + references
                       # (built incrementally on every event; queryable via `dejavue recall`)
```

Notes:
- `.dejavue/` (not `.joker/`) — avoids collision with joker-mcp's existing `.joker/cache/` directories (foreman perspective §10).
- `fts.db` is sqlite stdlib in Python 3 — zero install. FTS5 ships with system sqlite on Arch / most modern Linux.
- Add `.dejavue/fts.db` and `.dejavue/*.tmp` to `.gitignore` (rebuildable from JSONL). Commit the JSONL + markdown files only.

## Command surface (v0.1)

| Command | Purpose | Notes |
|---|---|---|
| `dejavue init` | Create `.dejavue/`, scaffold files, install git post-commit hook. | Idempotent. Won't overwrite existing files. |
| `dejavue start --agent <name> --goal <text>` | Record session start event. | Foundation for `since <agent>`. |
| `dejavue changed <path> --summary <text>` | Record file change event. Auto-fired by git hook. | Manual for non-commit-shaped changes. |
| `dejavue decision <title> --reason <text>` | Append decision to decisions.md + timeline. | Worthiness gate enforced by caller. |
| `dejavue state --summary <text>` | Overwrite state.md with current snapshot. | Single source of "what's stable now". |
| `dejavue handoff --summary <text> --next <text>` | Write handoff.md. | Session-close artifact. |
| `dejavue context` | Print all four .md files + last 10 timeline entries. | Boot packet, equivalent to `joker_get_context`. |
| **`dejavue since <ref>`** | Delta over time. `<ref>` = agent name, commit hash, or ISO date. | **Killer command. New in v0.1.** |
| **`dejavue ingest`** | Scrape `.claude/`, CHANGELOG.md, ADRs, git log into timeline. | **One-shot, marker-gated. New in v0.1.** |
| **`dejavue recall <query>`** | FTS5 keyword search over timeline + decisions + state + handoff + references. | **FTS5 only in v0.1; semantic in v0.2.** |
| **`dejavue worthiness`** | Print the worthiness gate (capture/skip table) to remind the caller. | **Built-in pedagogy. New in v0.1.** |
| **`dejavue get <doc>`** | Direct fetch of one artifact (state, handoff, decisions, references/<name>). | **ctx-adopted. Tab-friendly direct path.** |
| **`dejavue list [--type events\|decisions\|references]`** | List available artifacts with paths. | **ctx-adopted. Discovery surface.** |
| **`dejavue annotate <doc> "note"`** | Append timestamped note to a doc without rewriting. | **ctx-adopted. Evolving doc updates.** |

13 commands. Bold = foreman additions beyond chatgpt's original 7; trailing three are ctx-pattern absorptions.

## Event schema (timeline.jsonl)

```jsonc
{
  "ts": "2026-05-13T12:34:56-05:00",     // ISO 8601 with offset
  "agent": "claude",                       // who recorded the event
  "event": "session_start|file_changed|decision|state_update|handoff|ingest|init",
  "branch": "main",                        // git branch at event time
  "commit": "a81f2cd",                     // git short hash at event time
  "summary": "...",                        // one-line human-readable
  // event-specific fields below
  "path": "src/auth.rs",                   // for file_changed
  "diff_stat": "+12 -3",                   // for file_changed (auto from git hook)
  "decision_title": "Use JSONL",           // for decision
  "decision_reason": "...",                // for decision
  "goal": "...",                           // for session_start
  "rejected_alternatives": [                // foreman addition §7: log dead ends
    {"option": "use SQLite blob", "reason": "merge conflicts on binary"}
  ]
}
```

Append-only. Never edit existing entries. Schema additions in future versions must be backward-compatible (new optional fields only).

## `dejavue since` semantics

The killer command. Three forms:

```
dejavue since 2026-05-10        # events on or after date
dejavue since a81f2cd           # events on or after commit's timestamp
dejavue since --agent codex     # events since codex's last session_start event
```

Output structure:

```
Since <ref>:

Git delta:
  <git log --oneline ref..HEAD>
  <git diff --stat ref..HEAD>

Timeline events (N):
  <newest first, formatted for human read>

Decisions made (N):
  <appended in window>

State transitions:
  <if state.md was updated in window, show diff>

Handoffs in window:
  <list>

Topics (FTS5 keyword cluster):
  <top 5 keywords by tf-idf over the window's events>
```

This is the deliverable that nothing else gives you cleanly — git tells you the *what*, dejavue tells you the *why and intent* during the window.

## `dejavue ingest` strategy

One-shot scrape. Marker file prevents re-runs. Order of operations:

1. `git log --since=1.year` → emit one `file_changed` event per commit (path, diff_stat, commit message as summary).
2. `.claude/CLAUDE.md`, `AGENTS.md`, `.cursorrules` (if present) → emit one `decision` event each with title="Agent instruction file", reason=file contents (truncated 500 chars).
3. `CHANGELOG.md` (if present) → emit one `decision` per release entry.
4. `docs/decisions/*.md` or `docs/adr/*.md` (if present) → emit one `decision` per file.
5. `README.md` → emit one `state_update` event with summary=first paragraph.
6. Create `.dejavue/ingested.lock` with timestamp + ingested-file list.

`dejavue ingest --force` re-runs (warns about duplicate events).

## Hook integration

`dejavue init` writes `.git/hooks/post-commit`:

```bash
#!/usr/bin/env bash
# dejavue auto-capture
exec dejavue changed --auto --commit "$(git rev-parse HEAD)"
```

`dejavue changed --auto --commit <sha>` reads:
- `git show --stat <sha>` for diff_stat
- `git log -1 --format=%s <sha>` for summary
- emits one `file_changed` event per touched file

This is the entire auto-capture loop for v0.1. SessionStart / Stop hooks for Claude Code are v0.2.

## Recall surface (v0.1: FTS5)

`dejavue recall <query>` does:

1. Rebuild `fts.db` if `.dejavue/*.md` or `timeline.jsonl` is newer than `fts.db`.
2. `SELECT * FROM events_fts WHERE events_fts MATCH ? ORDER BY rank LIMIT 10` against the FTS5 virtual table built over:
   - timeline.jsonl event summaries + decisions + goals
   - decisions.md entries
   - state.md content
   - handoff.md content
3. Print results with `ts`, `event`, `summary`, and source-file hint.

v0.2 path (post embedder fix): add `--semantic` flag that calls `joker_search_knowledge` over the same content via kcs.db. FTS5 stays as fallback when embedder is degraded.

## Worthiness gate (built-in pedagogy)

`dejavue worthiness` prints:

```
Worthiness gate — only persist if:

CAPTURE                                    SKIP
─────────────────────────────────────────  ─────────────────────────────────────────
Decision changes architectural direction   Style preferences (let .editorconfig do it)
Constraint non-obvious from the code       Things git diff already shows
Blocker requiring external context         "Ran tests, passed"
Handoff context next agent must know       Per-file mechanical edits
Dead end + why it was rejected             LLM reasoning steps
Cross-cutting invariant ("X never depends  Routine commits
  on Y")

Rule of thumb: if removing this memory wouldn't confuse a future agent reading
the code + git log, don't write it.
```

Every command should also print this on first use via a `.dejavue/.first-use` marker (one-time message). Adoption depends on dev/agent picking up the right discipline early.

## Risk register

1. **FTS5 not available on the system sqlite.** Mitigation: probe at `dejavue init`; fall back to plain LIKE-search if missing. Print warning at recall time.
2. **Git hook clobber.** Mitigation: `dejavue init` checks if `.git/hooks/post-commit` exists with non-dejavue content; refuses to install unless `--force` passed.
3. **Ingest creates duplicate events on re-run.** Mitigation: `ingested.lock` marker + `--force` for re-runs with warning.
4. **No `git` in environment.** Mitigation: dejavue gracefully degrades (skips git-info fields in events, prints warning at init).
5. **Embedder bug "magically fixed" mid-session.** Mitigation: v0.1 doesn't depend on it. v0.2 will add `--semantic` flag.
6. **Dejavue + .jagent run in same repo.** Mitigation: v0.1 doesn't read or write to `.jagent/`. v0.2 will add the migration tool. They coexist harmlessly until then.
7. **Drift from chatgpt's vision.** Mitigation: keep chatgpt's `context.md`/`continued.md` in repo unchanged. `foreman_perspective.md` is dissent on record. v0.1 is the synthesis.
